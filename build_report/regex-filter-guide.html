Basic Usage

Just type any text to find rows containing that text (case-insensitive):

tcp Shows all rows containing "tcp" anywhere
iperf Shows rows with "iperf"
success Shows successful tests
failed Shows failed tests
200 Shows rows containing "200"

Pattern Matching with Regex

Find Multiple Terms (OR)

Use the pipe | character:

tcp|udp              ‚Üí Rows with "tcp" OR "udp"
200|500|1000         ‚Üí Rows with 200, 500, or 1000
success|passed       ‚Üí Successful or passed tests
failed|error|timeout ‚Üí Any problems

Wildcard Matching

The .* matches any characters:

tcp.*200            ‚Üí "tcp" followed by "200" anywhere after
protocol.*tcp       ‚Üí "protocol" followed by "tcp" somewhere
success.*iperf      ‚Üí Successful iperf tests
test.*\.log         ‚Üí "test" followed by ".log" anywhere

Start and End Matching

^test               ‚Üí Rows starting with "test"
\.log$              ‚Üí Rows ending with ".log"
^test.*\.log$       ‚Üí Starts with "test" and ends with ".log"

Real-World Examples

Example 1: Find all failed tests

Type: failed|error|timeout

Result: Shows only rows with failures, errors, or timeouts

Example 2: Find TCP tests with 200Mbps

Type: tcp.*200

Result: Rows with "tcp" followed by "200" anywhere after

Example 3: Find high-performance tests (500+ Mbps)

Type: [5-9]\d{2}Mbps

Result: Matches 500Mbps, 750Mbps, 999Mbps

Example 4: Find successful iperf tests

Type: iperf.*success

Result: Rows with "iperf" followed by "success"

Example 5: Find any throughput measurement

Type: \d+Mbps

Result: Matches 200Mbps, 500Mbps, 1000Mbps, etc.

Keyboard Shortcuts

Key Action

  • Green text: Results found (e.g., "Showing 12 of 203 rows")
  • Red text: No matches found
  • Blue border: Input is focused
  • Red border: Invalid regex (will use plain text search)

Tips & Tricks

üí° Tip 1: Start Simple, Then Add Complexity

Begin with basic text like tcp, then refine to tcp.*success, then tcp.*success.*200

üí° Tip 2: Use OR for Quick Comparisons

200|500|1000 lets you see multiple values at once

üí° Tip 3: Case Doesn't Matter

TCP, tcp, and Tcp all match the same rows

üí° Tip 4: Check the Match Count

If the count is 0, your pattern might be too specific

üí° Tip 5: Press Escape to Reset

Quick way to clear filter and see all data again

Common Mistakes to Avoid

‚ùå Don't Use Quotes

Wrong: "tcp" (searches for literal quotes)

Right: tcp

‚ùå Don't Forget .* Between Terms

Wrong: tcpsuccess (looks for "tcpsuccess" together)

Right: tcp.*success (tcp followed by success)

‚ùå Don't Forget to Escape Special Characters

Wrong: file.log (matches "fileXlog" too)

Right: file\.log (matches only "file.log")
