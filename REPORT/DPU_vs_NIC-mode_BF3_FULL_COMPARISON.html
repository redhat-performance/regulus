Key Insights

1. Topology Makes the Difference

  • Internode (cross-host): DPU dominates with 86.7% win rate across all test types
  • Intranode (same-host): NIC-mode dominates with 77.8% win rate
  • DPU hardware offload excels for network traffic but adds overhead for local traffic

2. DPU's CPU Efficiency Advantage

  • 34.5% fewer busy CPUs for internode workloads (39 vs 59.5 average)
  • 78% better CPU efficiency - delivers 6.99 Gbps/CPU vs NIC-mode's 3.93 Gbps/CPU
  • Hardware offload frees host CPUs for application workloads
  • Saves 9-33 busy CPUs per workload, critical in CPU-constrained environments
  • Superior CPU efficiency even when throughput is similar to NIC-mode

3. DPU Advantages for Internode Traffic

  • Consistent performance gains across all packet sizes (512B to 134KB)
  • Strongest advantage with medium CPU allocations (29 CPUs: saves 23-30 busy CPUs)
  • Best result: 329.80 Gbps with just 2 guaranteed CPUs using 44.8 busy CPUs
  • Superior performance for both stream and request-response workloads

4. NIC-mode Advantages for Intranode Traffic

  • 74% faster for intranode stream workloads (497 vs 286 Gbps average)
  • 87% faster for intranode request-response (2.9M vs 1.5M trans/sec)
  • Benefits from kernel-level optimizations for same-host communication
  • No DPU overhead for local pod-to-pod traffic

Recommendations

Choose DPU for:

  • Internode workloads - Cross-host network traffic (+16.6% throughput, -34.5% CPU)
  • CPU-constrained environments - Save 20-30 busy CPUs per workload
  • Multi-threaded applications - 32+ threads performing network I/O
  • Maximum CPU efficiency - Up to 14.78 Gbps/CPU vs NIC-mode's 6.84 Gbps/CPU
  • Small to large packet sizes - Consistent gains across 512B-134KB
  • Request-response patterns - 53% advantage for internode RR workloads
  • Cost optimization - Lower CPU consumption reduces infrastructure costs

Choose NIC-mode (BF3-Bond) for:

  • Intranode workloads - Same-host pod-to-pod communication (+74% throughput)
  • Mixed workloads - Combination of local and network traffic
  • Simplicity - No DPU hardware or configuration required
  • High intranode throughput - Up to 676 Gbps for local traffic
  • CPU abundance - When CPU resources are not a constraint
